# Lnwtermgame — Design Context

## Design Context

### Users
- **Age range:** 13–35 years old — teens through early working adults
- **Primary segments:**
  - **Competitive/Hardcore Gamers** who need game currency or items instantly to get back to competing (Valorant, FiveM, etc.)
  - **Casual Gamers** who want convenient top-up across multiple payment channels (credit cards, e-wallets, PromptPay)
- **Price sensitivity:** Very high. Users actively compare prices across platforms. Promotions like "10% cashback" and "50% bonus credit for new users" are key decision drivers. **Value-for-money is the primary purchase motivator.**
- **Context of use:** Often switching from a mobile game app to top up mid-session. Speed and minimal friction are critical. Desktop users also exist (PC gamers), but mobile-first is essential.

### Brand Personality
**Fast · Secure · Valuable** *(+Smart — AI-powered assistance)*

- **Fast:** Payment to delivery in seconds. The interface should feel instant and responsive.
- **Secure:** Financial transactions and game account IDs demand trust. Every interaction should reinforce safety.
- **Valuable:** Users should feel they made a smart choice. Promotions, rewards, and clear pricing build this perception.
- **Smart:** AI game assistant and intelligent recommendations position the platform as innovative and forward-thinking.

### Emotional Goals
| Emotion | How to achieve it |
|---|---|
| **Trustworthy & Secure** | Clear transaction statuses, security badges, consistent professional UI, no visual clutter |
| **Fast & Instant** | Minimal steps to purchase, auto-delivery indicators, progress feedback, snappy animations |
| **Smart & Innovative** | AI assistant feature, intelligent product suggestions, modern design patterns |
| **Rewarding** | Visible promotions, cashback indicators, loyalty signals, "you saved X" confirmations |

### Aesthetic Direction
- **Visual system:** Neo-Brutalism Lite — bold black borders, offset shadows, candy-bright accent colors on a clean white/light gray canvas
- **Color palette:** Pink (#FF6B9D), Yellow (#FFD93D), Teal (#4ECDC4), Mint (#95E1D3) as accents. Black for structure. White/gray (#f9fafb) for backgrounds. Trust sections use muted professional tones.
- **Typography:** Poppins (primary), Noto Sans Thai (Thai content), system fallbacks per locale
- **Theme:** Light mode primary. Dark mode is a future consideration (gamer audience preference).
- **Tone:** Energetic but not chaotic. Bold but not aggressive. Fun but trustworthy.

### References
| Reference | What to learn from it |
|---|---|
| **Codashop** | Ultra-short purchase flow, no unnecessary steps, smooth and focused checkout |
| **Razer Gold** | Gamer-centric identity, premium wallet/rewards system, strong brand presence |
| **SEAGM** | Clear product categorization across games, digital cards, and mobile recharge |

### Anti-References
| Avoid | Why |
|---|---|
| **Corporate/rigid bank-like UI** | Kills the gaming energy and fun. Too formal = boring for the audience. |
| **Spammy/gambling vibes** | Flashing colors, cluttered banners, and chaotic layouts destroy trust instantly. |
| **Clunky checkout flows** | Long forms and unnecessary steps cause abandonment. Top-up should be 2-3 clicks. |

### Design Principles
1. **Speed is respect.** Every interaction should minimize time-to-completion. If a step can be removed, remove it. If a transition can be faster, make it faster. Gamers alt-tab to top up — don't make them wait.
2. **Trust through clarity.** Use clear visual hierarchy, consistent patterns, and explicit status indicators. Never leave users guessing about their transaction state. Use both color AND text for all system messages.
3. **Bold, not loud.** The neo-brutalism aesthetic should feel intentional and confident, not chaotic. Strong borders and bright accents create energy; visual clutter and animation overload create anxiety.
4. **Value is visible.** Discounts, savings, and promotions should be immediately scannable. Users compare prices — make the value proposition impossible to miss.
5. **Mobile is the main stage.** Design for thumb-reach, touch targets, and small screens first. Desktop is the expanded version, not the other way around.

### Accessibility Baseline
- **WCAG AA compliance** as minimum target
- **Touch targets:** Minimum 44×44px for all interactive elements (buttons, inputs, game UID fields)
- **Contrast:** Text must meet AA contrast ratios against all backgrounds
- **Keyboard navigation:** Full support for Tab-based navigation on desktop (PC gamers use keyboard-first)
- **Status messaging:** All error/success states must use BOTH color AND text — never color alone
- **Reduced motion:** Respect `prefers-reduced-motion` for all animations
- **Mobile-first responsive:** All layouts must work at 375px minimum width

### Technical Stack
- **Framework:** Next.js 16 + Turbopack
- **Styling:** Tailwind CSS with custom design tokens
- **Fonts:** Poppins (latin), Noto Sans Thai, Noto Sans SC/JP/KO/Devanagari (i18n)
- **Animation:** Framer Motion
- **Icons:** Lucide React
- **i18n:** next-intl (TH, EN, ZH, JA, KO, HI)
